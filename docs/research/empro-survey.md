# EMPRO 機能調査レポート

**調査日**: 2026-06-14
**対象**: emprocloud.jp(株式会社YOROZUYA で契約中の空アカウント)
**目的**: 人材紹介エージェント向け CRM/ATS の機能を網羅的に把握し、Maira の agency 側に取り込むべき機能を整理する

**生データ**: `/tmp/empro-survey/empro-survey-output/`(スクショ + JSON)

## 1. EMPRO とは

EMPRO(エンプロ)は **人材紹介エージェント向けの SaaS CRM/ATS**。職種に「CA(キャリアアドバイザー = 求職者担当)」と「RA(リクルーティングアドバイザー = 企業担当)」の役割概念がある業界特化型ツール。

Maira とのポジショニング差:

- **Maira**:求職者(20-30代)向け AI 採用エージェント(BtoC)+ 紹介エージェント向け管理画面(BtoB の agency 側)
- **EMPRO**:紹介エージェント向け管理画面に特化(BtoB のみ)

→ **Maira の agency 側が機能比較の対象**。BtoC 側は無関係。

## 5. Maira への実装推奨(優先度順)

### Phase A — 法令・基本機能(着手しやすく効果大)

1. **求人項目の労基法準拠拡張**
   - DB マイグレーション + UI:`業務内容（変更の範囲）` `就業場所（変更の範囲）` `試用期間中の待遇変更` `受動喫煙防止措置`
   - 既存 job_postings に列追加(暗号化対象に含めるか要検討)
2. **求職者プロフィールの拡張**
   - 「連絡方法希望」「メール配信」「エントリーサイト」「他社エージェント利用状況」
   - 暗号化対象に追加
3. **CSV インポート**(エクスポートは既にあり)
   - 求職者・求人

### Phase B — 営業管理(Maira を本格的な業務ツールに)

4. **ヨミ表(売上見込み管理)**
   - referrals に「ヨミ対象月」「売上見込み」「分配比率」「入社予定日」を追加(既にあれば確認)
   - 月別予測ビュー
5. **目標設定 + スコアカード**
   - 月次目標(担当者別)→ ダッシュボードに進捗バー
6. **カンバンビュー**
   - 選考プロセスのステージごとカンバン表示

### Phase C — コミュニケーション(差別化)

7. **メール送信統合**(Gmail/Outlook OAuth)
   - Resend は招待リンク程度なので、本格運用には OAuth で「担当者本人のメールから送信」できる仕組みが必要
8. **LINE 公式アカウント連携**(求職者と LINE で会話)
   - LINE Messaging API
   - 求職者ごとの友達追加リンク
   - 担当CAごとの受信箱
9. **面談予約(カレンダー連携)**

### Phase D — 高度な機能(後回し)

10. **AI自動入力**(求人URLから求人情報を抽出)
11. **カスタムビュー保存 / 詳細フィルター**
12. **MA(Eメール/LINE一斉配信)**

## 6. Phase 2 で深掘りした内容(登録モーダル + マーケティング + 目標設定)

Phase 2 では各エンティティの「登録モーダル」を開き、フォーム項目を抽出した(保存はせずキャンセル)。

### 6.1 求職者 新規登録フォーム(56フィールド)

セクション構成:

- **基本情報** — 姓\* / 名\* / 姓カナ / 名カナ / 生年月日 / 年齢 / 性別 / 国籍 / 配偶者なしフラグ / 郵便番号 / 都道府県 / 市区町村 / 番地 / 建物名・部屋番号
- **連絡先情報** — 電話番号①/② / メールアドレス①/② / 連絡方法に関する希望(textarea, max 1000) / メール配信ON/OFF
- **面談・ステータス情報** — 面談実施日 / 面談所感 / **推薦コメント** / **求職者ステータス\*** / ステータスメモ / **求職者クローズ理由** / クローズ理由詳細
- **就業情報** — 現在の雇用形態 / 現在年収(万円) / 経験業種 / 経験職種 / 最終学歴 / 学歴(詳細) / 保有資格・スキル
- **希望条件** — 希望業種 / 希望職種 / 希望勤務地 / 希望転職時期 / 希望年収 / 転職理由 / 希望条件詳細

→ 注目: **推薦コメント**(エージェントが企業に推薦する際の文章)はMairaにも欲しい。

### 6.2 企業 新規登録フォーム(28フィールド)

- **基本** — 企業名\* / 企業名カナ\* / 業種 / 電話番号 / ウェブサイトURL / 住所(郵便番号〜建物名)
- **企業情報** — 代表者名 / 従業員数 / 設立日 / 資本金 / 年間売上高 / 事業内容
- **ATS連携(チェックボックス + 各サービスのURL欄)**:
  - HRMOS / HERP Hire / ジョブカン採用管理 / sonar ATS / HRBrain採用ソリューション / Talentio
  - 「ATS(採用管理システム)を使用」フラグでON

→ 注目: **6種類の主要ATSとの連携URL登録**機能。Maira でも企業データに「採用ページURL」を持たせて自動同期する余地あり。

### 6.3 求人 新規登録フォーム(55フィールド、必須項目を網羅)

必須項目(`*`):
**企業名 / 求人名 / 職種 / 業務内容 / 就業場所(都道府県) / 勤務時間 / 休憩時間 / 休日休暇 / 受動喫煙防止措置**

主要項目:

- 採用担当者(複数選択可) / 担当RA / 求人ステータス / クローズ理由 / 求人URL
- ポジション / 雇用形態 / 雇用形態詳細 / 就業場所詳細
- 年齢制限(下限/上限) / 転職回数制限 / 求める最終学歴 / 応募資格
- 年収レンジ(下限/上限) / 給与詳細 / 残業手当 / 社会保険 / 福利厚生 / 受動喫煙詳細
- 選考フロー / 面接情報 / 面接対策 / 求人のお薦めポイント

→ 注目: **必須9項目は労基法/職業安定法準拠**。Maira の求人スキーマ拡張の優先順位はこの順。

### 6.4 選考プロセス 新規登録フォーム(21フィールド)

セクション構成:

- **基本情報** — 求職者\*(検索) / 求人\*(検索) / 担当CA / 担当RA / 不採用理由
- **選考プロセス** — 選考ステータス\*(デフォルト「求人打診」) / 面接確定日 / ステータスメモ
- **ヨミ情報** — ヨミステージ / ヨミ対象月 / 売上見込(円) / 入社予定日 / **売上分配比率(CA/RA、合計100%必須)**

**注目: 「面接確定日の日時を起点に面接後フォローMAが自動配信されます」**

→ 重要パターン:

1. 「ステータス変更/特定日時」をトリガに **自動メール配信**(MA連携)
2. 売上の **CA/RA 50:50 分配**(変更可能、合計100%チェック)→ アドバイザー別売上計算ロジックの基盤

### 6.5 ダッシュボード「目標設定へ」リンクの行き先

`/settings?section=dashboard` の「ダッシュボード・ヨミ表設定」セクション。**5機能カード**で構成:

1. **選考ステータス月次目標** — 各選考ステータスごとの月次目標数(打診100件、面接30件、内定5件 のように設定)
2. **選考ステータス転換率** — ステータス間の転換率(打診→面接 30%、面接→内定 17% のような実績計算用)
3. **営業日カレンダー** — 営業日と休日を設定(月次目標の日割り計算に必要)
4. **月次売上目標** — 担当者別の月次売上目標 + RA/CA分配比率
5. **ヨミステージ** — ヨミ表(売上予測)のステージ定義・並び替え

→ これが「KPI ダッシュボード」が機能するための前提設定。Maira にも同等の「組織設定」UI が必要。

### 6.6 マーケティング(Eメール MA / LINE MA)

クリックすると **利用同意モーダル**が表示される(別画面ではない)。同意項目:

**Emproが許容する利用範囲**:

- まとめ送信目的の登録 / 営業連絡や面接案内などの個別通知 / 求人マッチに基づく案内 / ワンクリック解除手段の提供 / LINEブロック等の拒否ユーザー除外 / 配信頻度の自制

**お客様にお願いしたいこと**:

- 法令の遵守 / LINE公式ガイドラインの遵守 / LINEアカウントの認証維持 / 安全管理対策の徹底

→ MAは法令対応・スパム対策で慎重設計。Maira で実装する場合も**配信停止リンク必須**・**ブロック検知**を組み込む必要あり。

## 8. Phase 3: 詳細画面・LINE連携・ユーザー管理(データ入り状態で再撮)

**両エンティティ共通**:

1. リスト画面と詳細を同時に見られる(コンテキスト維持)
2. キーボード操作で行を切り替えやすい

**アコーディオンセクション(9個)**:

1. 基本情報
2. 連絡先情報
3. 面談・ステータス情報
4. 経歴
5. 希望条件
6. 登録情報(担当CA、受付日、ステータス、ステータスメモ等)
7. **求人一覧** (N件) — この求職者に紹介中の求人
8. **アクティビティ履歴** (N件、「+作成」ボタン付き) — 電話/メール/面談の対応履歴
9. **添付ファイル** (N件、「+アップロード」ボタン付き) — 履歴書/職務経歴書

→ Maira agency 側との対応:

- 「アクティビティ履歴」=Maira の client_interactions(既存)
- 「添付ファイル」=**Maira に未実装**(履歴書 PDF アップロード機能)

- 求人フォームに「採用担当者(複数選択可)」フィールドがあったので、**多対多関係**を持っている

### 8.4 設定 - LINE連携設定(展開後の詳細)

LINE公式アカウント連携の2モード:

**A) LINE公式アカウントをお持ちでない方** — セットアップ手順を表示:

**B) すでにLINE公式アカウントをお持ちの方** — Channel Access Token / Channel Secret 入力フォーム

- LINE Messaging API の Channel Access Token / Channel Secret を組織単位で保存(暗号化必須)

### 8.5 設定 - ユーザー管理(展開後の詳細)

### 10.4 ダッシュボード(目標設定後の実態)

ユーザーがダッシュボード・ヨミ表設定で目標を入れた後の表示:

**ヘッダー**:

- 「**6月 営業日実績 45% 10/22営業日**」(プログレスバー)
- ナビレベルのフィルター: CA/RA / 集計軸 / 担当CA / 選考ステータス更新日

**「6月 営業日実績」スコアカードセクション**:
横並びの選考ステータス**10段階**:

```
求人打診 → 応募承諾 → 書類推薦 → 書類選考OK → 1次面接 → 2次面接 → 最終面接 → 内定 → 内定承諾 → 不合格
```

各カードに:

- **実績**: 数値(現在の進捗)
- **営業日目標**: その時点でのあるべき値(月次目標 × 営業日経過率)
- **差分**: 実績 - 営業日目標(赤字 -1, など)
- **達成率**: パーセント

「不合格」も独立した目標として扱える(離脱率管理用)。

**「6月 担当者別実績」セクション**:

- 「チャート + ランキング」表示切替
- 担当者ごとの達成率比較

**「求職者一覧」セクション**:

- 面談実施日のフィルター
- 「すべて開く」ボタン

→ Maira の reports セクションは個別に分かれているが、EMPRO はこれを **「営業ダッシュボード」として統合**している。日次の動向を1画面で見せる設計。

3. **変数挿入型エディタ**: 候補者・担当CA・組織名の変数をクリック挿入
4. **配信上限の表示**: 「0 / 5,000」のような月次上限を可視化(課金/契約プランに紐付け)
5. **β版バッジ**: 機能の安定度を明示し、ユーザーの期待値を調整
