# FAQ / 用語集 / トラブルシュート

NotebookLM で「これってどういう意味?」「○○が動かない」を質問された時に AI が答えられるよう、用語と FAQ をまとめます。

## FAQ(エージェント企業の経営者・担当者向け)

### Q. テスト導入中のデータは、本契約後そのまま使えますか?

はい。テスト中に入力したデータは本契約後そのまま引き継がれます。解約時は CSV / JSON で一括エクスポート可能です。

### Q. 求職者への公開範囲はコントロールできますか?

求職者が個別に「連携解除を申請」で開示を停止でき、猶予期間経過で自動確定します。二段階解除設計により、業務中断のリスクを防いでいます。

### Q. LINE 連携はどの規模まで対応できますか?

LINE Messaging API の Light(5,000 通/月)から Pro まで全プランに対応。配信数の残量は Maira 内で可視化されます。

### Q. Google Meet / Zoom 連携で、カレンダーの既存予定は見えますか?

scope は最小限(calendar.events)で、Maira から作成した予定のみ編集・削除します。既存予定の読み取りはしません。

### Q. AI 利用量に上限はありますか?

月次で機能別の上限があり、アドオン契約で緩和できます。残量はダッシュボードで常時確認可能です。

### Q. サポートの連絡手段は?

メールおよびお問い合わせフォームから、1 営業日以内に一次返信します。

### Q. データはどこに保存されますか?

Supabase(クラウド DB、東京リージョン)に保存されます。会話履歴・キャリア情報等の機密フィールドは AES-256-GCM で暗号化されます。

### Q. 1 つの組織に何人まで登録できますか?

プランによります。Free プランでは admin 1 + advisor 2 程度、Premium プランで 50 名規模まで想定。詳細はお問い合わせください。

### Q. 既存の CSV データを移行できますか?

はい。`/agency/clients` の「CSV インポート」で日本語ヘッダーを AI が自動マッピング → 上限 200 行 / 8 MiB / 1 回。求人も同様にインポート可能。

### Q. 個人情報保護法・個人情報の取扱いは?

プライバシーポリシーで運営者の閲覧範囲を明示しています。暗号化と RLS により、運営者であっても日常的にユーザーデータを閲覧することはありません(保守 / 本人同意済の AI 処理 / 法令対応 を除く)。

### Q. 料金はいくらですか?

本ガイドでは金額を提示していません。資料請求フォームから問い合わせください。

### Q. 解約はいつでもできますか?

はい。Stripe Customer Portal から月次でいつでも解約可能。次回請求日まで利用継続、データエクスポートも可能です。

### Q. 求職者 1 人につきいくらかかりますか?

エージェント企業側の月額サブスクリプションのみで、求職者数による従量課金は基本ありません(LINE 配信数等は LINE プラン依存)。

### Q. オンプレミス版はありますか?

現状クラウド SaaS のみ。専用環境のご要望は別途ご相談ください。

### Q. API は公開していますか?

現状はアプリ内利用のみ。外部 API 公開のロードマップは検討中。

## FAQ(求職者向け)

### Q. 利用は無料ですか?

求職者の基本機能は無料です。AI 機能には月次の利用上限があります。

### Q. 連携したエージェントに何が見られますか?

履歴書・職務経歴書・希望条件・プロフィール、キャリア棚卸し結果(wants と user_facts のみ、内面情報は除外)。

### Q. エージェント連携を解除したら、すぐ情報が見られなくなりますか?

「解除を申請」した後、猶予期間(エージェント設定の N 日)を経過するか、エージェントが「即時承認」すると停止します。猶予期間中は引き続き開示されます。

### Q. 履歴書 PDF を他社にも提出できますか?

はい。PDF はあなたの所有物です。Maira からダウンロードして自由に使えます。

### Q. アカウントを削除したらデータはどうなりますか?

あなたのアカウント情報・履歴書・棚卸し結果は削除されます。エージェント側に紐づいていた client_records はエージェントの業務記録として残りますが、あなたの個人情報は閲覧できなくなります。

## 用語集

### 一般用語

- **クライアント / 求職者 / seeker**: Maira を使う一般ユーザー。転職を考えている人。
- **エージェント / advisor / 担当者**: 転職エージェント企業の担当者(一般メンバー)。
- **管理者 / admin**: エージェント企業の管理権限を持つメンバー。組織設定 / 課金 / メンバー管理が可能。
- **組織 / organization**: エージェント企業の単位。1 ユーザー = 1 組織所属。
- **連携 / link**: 求職者とエージェントが Maira 上で情報を共有する関係。

### 機能用語

- **CRM**: Customer Relationship Management。Maira では「クライアント管理」を指す。
- **referrals**: 紹介。エージェントが求職者を求人に紹介した記録。
- **placements**: 成約。紹介が成立して入社した記録。
- **introduced / interviewing / offered / placed**: 応募の状態遷移。
- **沈黙顧客**: 一定期間(30 / 60 / 90 日)対応がない顧客。Maira が自動検出。
- **Daily ダイジェスト**: 朝 8 時に admin に届くまとめメール。
- **キャリア棚卸し**: 求職者が AI と対話して自分の強み・価値観・希望条件を整理する機能。
- **面接シミュレーター**: 求職者が 1 人で面接練習する機能(音声 I/O 対応)。
- **会議録音 → AI 履歴書**: 面談録音から自動的に履歴書・職経歴書を生成する機能。
- **推薦文**: エージェントが求人企業に提出する候補者紹介文。

### LINE 関連

- **公式 LINE OA**: LINE Official Account。エージェント企業が運営する公式アカウント。
- **friend**: 公式 LINE の友達。Maira では line_user_links で管理。
- **連携コード**: 求職者がエージェントの公式 LINE と Maira アカウントを紐付けるための 6 桁コード。
- **MA**: Marketing Automation。Maira では LINE 配信シナリオ自動化を指す。
- **シナリオプリセット**: LINE MA で予め用意された 7 種類の配信タイミング(ウェルカム / リマインド 等)。
- **一斉配信**: 友達全員(または対象を絞って)に同じメッセージを配信する機能。
- **Flex Message**: LINE の構造化メッセージ。求人カードや面談候補日提案で使用。
- **Reply Token**: LINE の返信 API トークン。受信から 30 秒以内なら無料で返信可能。

### 技術用語

- **RLS**: Row Level Security。Supabase の行レベルアクセス制御。組織単位でデータ分離。
- **AES-256-GCM**: 認証付き対称暗号方式。Maira の暗号化対象フィールドで使用。
- **HMAC-SHA256**: Webhook の署名検証アルゴリズム。LINE / Stripe / Zoom 等で使用。
- **Webhook**: 外部サービスから Maira への通知 URL。
- **OAuth**: Google / Zoom 連携時の認可プロトコル。
- **Whisper**: OpenAI の音声認識 API。Maira で文字起こしに使用。
- **Claude**: Anthropic の AI モデル。Maira では Sonnet 4.6 を主に使用。

### 設定用語

- **notification_prefs**: 各メンバーの通知設定(jsonb)。種類別 ON/OFF + メール全体マスター。
- **email_enabled**: メール通知の全体マスター。OFF にすると全種類のメール停止。
- **daily_digest**: 朝のダイジェストメール key。
- **revoke_grace_days**: 連携解除の猶予期間(日数)。
- **avatar_storage_path**: ユーザーアバター画像の保存パス。

## トラブルシュート

### ログインできない

1. メールアドレス / パスワードが正しいか確認
2. 「パスワードを忘れた」リンクで再設定
3. それでもダメな場合はサポートに連絡

### メールが届かない(招待 / 通知)

1. 迷惑メールフォルダを確認
2. ドメイン `@maira.pro` を受信許可リストに追加
3. それでも届かない場合は、フォームから連絡

### LINE 連携コードが効かない

- 連携コードは発行から 1 時間で失効
- 大文字小文字を正確に入力
- 既に別の Maira アカウントと紐付けられていないか確認

### 面談予約が確定しない

- 求職者がタップした際に「セッション切れ」エラーが出る場合、Flex Message が古い可能性
- エージェントから候補日を再送

### Google / Zoom 連携で「アクセスをブロック」エラー

#### 「redirect_uri_mismatch」

- Google Cloud Console で OAuth redirect URI に `https://app.maira.pro/api/integrations/google/callback` を追加

#### 「アプリはGoogleの審査プロセスを完了していません」

- アプリが「テスト中」ステータス
- 解決: Google Cloud Console → OAuth consent screen → テストユーザー → メール追加
- 本番公開後は警告なし

#### 「Manual linking is disabled」

- Maira 内部で linkIdentity を呼んでいた旧バグ。最新版で修正済(connect ルートに切替済)。

### Daily ダイジェストが来ない

1. 個人設定 → 通知設定 で email_enabled + daily_digest が ON か
2. メール受信フォルダで「【Maira 朝のダイジェスト】」を検索
3. 集計が全件 0 の日は「平和な朝」として送信されない(仕様)

### AI 残数が減らない / 増えない

- 2026-06 以前のバグ:CHECK 制約で半数の kind が記録漏れ → 修正済
- それでも違和感がある場合はサポートに連絡

### 画面が真っ白になる / 古い表示が残る

1. ブラウザを最新版にアップデート
2. シークレットウィンドウで再アクセス(キャッシュ回避)
3. キャッシュとクッキーをクリア

### LINE 一斉配信ボタンが押せない

- LINE Channel が未連携の可能性
- `/agency/line/settings` で接続を確認

### CSV インポートが失敗する

- 200 行 / 8 MiB の上限を超えていないか
- ヘッダー行が含まれているか
- 必須列(求人: 会社名 / 職種、クライアント: 名前)が入っているか
- 文字コードは UTF-8 推奨

### サイドバーが崩れた

- サイドバー編集 → 「初期化」で工場出荷状態に戻せます

## サポートへの問い合わせ前に確認

1. **画面のスクリーンショット**: エラーが見える状態で
2. **時刻 / ユーザー名 / 操作内容**: 「2026-06-22 14:00 頃、求人を新規作成しようとして失敗」など
3. **ブラウザの種類とバージョン**: Chrome 130 / Safari 17 等
4. **ブラウザの開発者ツール(DevTools)**: Console タブのエラーメッセージ

これらを添えてフォーム / メールから送ると、迅速に対応できます。
