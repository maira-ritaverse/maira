# 求職者向け 利用ガイド

求職者(seeker)が Maira を使う際の機能を網羅したマニュアルです。
Maira は「自分でキャリアを整理し、AI と一緒に進める」転職プラットフォームです。
スマートフォンの公式 LINE / Web 両方から使えます。

## はじめての登録

### サインアップ

1. `https://www.maira.pro/signup` にアクセス
2. メールアドレス + パスワードを入力
3. メール確認 → ログイン

### Google ログイン(任意)

- ログイン画面で「Google で続行」
- Google アカウントで認証 → 自動でサインアップ

### プライバシーポリシーへの同意

初回ログイン時にモーダルが表示されます。同意しないと機能が使えません。

### オンボーディングツアー

初回はダッシュボードで主要機能の使い方ツアーが自動起動。後で「設定 → オンボーディングツアー」から再表示可能。

## ダッシュボード(/app)

求職者画面のトップページ。

- **AI 利用残数**: 今月使える AI 機能の枠
- **次の面談**: 直近の予約面談
- **未読通知**
- **直近の AI 推薦**
- **応募中の状況**
- **自分のタスク**
- **状態別カード**(empty / starter / active で切り替え)

## キャリア棚卸し(/app/career)

AI と雑談しながら、自分の強み・価値観・希望条件を構造化する機能。

### 流れ

1. 「新しく棚卸しを始める」をクリック
2. AI(Claude)とチャット形式で会話
3. 質問に答えていくと、AI が構造化(キャリアプロファイル)を作成
4. 結果ページで「自己理解」「強み」「価値観」「希望条件」を確認

### 再診断

- 現在の棚卸し結果を上書きして再度実施可能
- 履歴書 / 職務経歴書の AI 下書きで活用される

### データの暗号化

会話履歴・棚卸し結果はサーバー側で AES-256-GCM 暗号化。

## 履歴書(/app/resumes)

厚労省推奨様式に沿った構造化履歴書を作成・管理。

### 新規作成

- 月 5 件まで(ブーストチケットで +10 件 × 3 ヶ月)
- 既存履歴書の複製は枠を消費しない

### 各項目の入力

- 氏名 / 連絡先 / 学歴 / 職歴 / 資格 / 志望動機 / 自己 PR 等

### AI 下書き

各項目で「AI で下書き」ボタン → キャリア棚卸し結果を元に Claude が文章生成。

### PDF ダウンロード

詳細画面の「PDF をダウンロード」ボタン。Puppeteer で生成。

### 証明写真の AI 補正

- 写真をアップロード
- AI で背景白抜き + 補正
- 月 5 枚まで(アドオン契約で 30 枚)

## 職務経歴書(/app/cvs)

履歴書と同じく構造化 + AI 下書き + PDF 対応。

### 主要項目

- 要約 / 職務経歴 / プロジェクト経験 / スキル / 資格 / 自己 PR

### AI 下書き

各項目で AI 生成可能。月 20 件のハード上限。

## 応募管理(/app/applications)

自分で管理する応募状況。

### 自分応募 vs エージェント経由

- **自分で応募**: 自分で直接登録
- **エージェント経由**: notes に「○○エージェント 経由」が含まれる行(ハンドシェイクアイコン付きバッジ)

### 状態管理

- 書類選考中 / 面接調整中 / 面接実施 / 内定 / 入社 / 見送り

### 期限管理

各応募にタスクや期限を設定すると、ダッシュボードに表示。

## AI 求人推薦(/app/job-recommendations)

連携中のエージェントが持つ求人の中から、AI が「あなたに合う求人」を推薦。

### 操作

- 推薦された求人カードを閲覧
- 「興味あり」または「応募を依頼」をタップ
- エージェント側に in-app + メール通知が飛ぶ

### 上限

- 月 20 件(アドオンで 200 件)

## 面接シミュレーター(/app/interview)

1 人で面接練習が可能。

### 流れ

1. 「新規セッション開始」をタップ
2. AI 面接官(Claude)が質問
3. **音声 I/O 対応**: マイクで回答 / AI 音声で読み上げ
   - iOS Safari でも動作(SpeechRecognition + speechSynthesis)
4. 回答後 AI フィードバック(良い点 / 改善点)
5. セッションは履歴に保存(暗号化)

### 練習モード

- 一般面接 / 技術面接 / 志望動機特化 等

## メッセージ(/app/messages)

エージェントとの個別メッセージ。LINE と独立した Web 内チャネル。

## 連携先(/app/connections)

自分とつながっているエージェント企業の一覧。

### 状態

- **招待中**: エージェントから招待が届いた状態
  - 「承認する」: 連携開始 → エージェントが履歴書等を閲覧可能に
  - 「拒否する」: 連携は始まらない
- **連携中**: 開示中
  - 「解除を申請する」: 猶予期間付きで解除申請
- **解除申請中**: 猶予期間内、申請の取下は不可
- **解除済**: エージェントから情報を見られない

### 開示範囲

連携中はエージェントが以下を閲覧可能:

- 履歴書 / 職務経歴書
- キャリア棚卸し結果(wants + user_facts のみ、内面情報は除外)
- 希望条件
- プロフィール

## LINE 連携

公式 LINE 経由で次のことが可能:

- 友達追加 → 連携コード入力で本人確認
- 求人カードの受信
- 面談候補日の選択(タップで Meet/Zoom URL 自動発行)
- メッセージ / スタンプ / 画像 のやりとり

### 友達追加

エージェントが運営している公式 LINE を友達追加 → エージェントから連携コードが送られる → 求職者がそれを入力で紐付け。

## 設定(/app/settings)

### トップ表示

- AI 利用残数(6 機能を 1 画面で確認)
- 各種設定へのリンク

### サブページ

- **プロフィール**: 表示名 + アバター画像
- **パスワード変更**
- **オンボーディングツアー再起動**
- **連携・利用状況**: Zoom / Google 連携、AI 残数の詳細
- **アカウント**: 登録メール確認 / アカウント削除

### アバター画像

- JPEG / PNG / WebP、2 MiB 以下
- 設定するとエージェント側のメンバー一覧等で顔写真表示

## アカウント削除

- 設定 → アカウント → 削除
- 確認モーダルで意思確認
- 本人のアカウント情報 + 履歴書 / 棚卸し等が削除
- エージェント側の client_records は残る(運用上の引き継ぎのため)が、開示は停止

## データのエクスポート

設定 → アカウント → データエクスポート要求 で全データを JSON ダウンロード可能(運営に依頼)。

## トラブル時

### LINE で連携コードが入らない

- 入力した連携コードが有効か(発行から 1 時間で失効)
- 大文字小文字を正確に入力(O / 0 と I / 1 は紛らわしい文字を除外済)

### 面接シミュレーターで音声が出ない

- ブラウザのマイク許可を確認
- iOS Safari の場合、設定 → Safari → サイトの設定 → マイク を許可

### 履歴書の PDF ダウンロードができない

- ポップアップブロックを解除
- ブラウザを最新版に更新
