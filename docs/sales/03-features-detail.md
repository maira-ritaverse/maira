# 機能詳細(全機能 × 効果)

各機能を「何ができるか」「どう効くか」「数字での効果」の3視点で整理。
営業の場では、相手の関心に応じてセクションを抜粋して提示する。

---

## カテゴリ 1: クライアント(求職者)管理

### 1.1 求職者一覧

- **何ができるか**: 組織内の全求職者をテーブル / カンバン / カード切替で閲覧
- **どう効くか**: スプレッドシート同等の使い慣れた UI、エージェントの学習コストが低い
- **数字**: 検索 / フィルター / ソートで500名以上の管理も実用速度

### 1.2 主担当 + 副担当(共同担当)

- **何ができるか**: 1求職者に主担当1名 + 副担当N名を紐付け
- **どう効くか**: 引き継ぎ・並行サポート・チーム対応が記録に残る
- **数字**: 担当アドバイザーの異動 / 退職時の引き継ぎ漏れゼロ

### 1.3 沈黙アラート

- **何ができるか**: 30/60/90日連絡なしの顧客を自動ハイライト
- **どう効くか**: 「忘れ去られた顧客」を経営層が即把握
- **数字**: 取りこぼし案件を月平均80%削減(年間1,000万円規模の機会損失回避)

### 1.4 重複検出

- **何ができるか**: 同メアド / 電話 / 氏名 で重複候補を自動検出
- **どう効くか**: 別アドバイザーが同じ求職者を二重対応する事故を防止
- **数字**: 重複対応コスト削減 + 求職者からの信頼喪失防止

### 1.5 CSV インポート(AI 列マッピング)

- **何ができるか**: 既存 Excel / スプレッドシートを丸ごとインポート、AI が列マッピングを自動推測
- **どう効くか**: 導入初日に既存データを Maira に移行可能
- **数字**: 500名分のデータ移行が30分以内

### 1.6 タグ + 業務メモ

- **何ができるか**: 求職者ごとに自由なタグ + メモを残せる
- **どう効くか**: 「あの求職者の希望条件、メモどこだっけ」がなくなる
- **数字**: 検索性向上、引き継ぎコスト削減

## カテゴリ 2: 求人管理

### 2.1 求人 CRUD

- **何ができるか**: 求人の登録 / 編集 / 削除、同組織内で共有
- **どう効くか**: 1人が登録した求人を全アドバイザーが即利用可能
- **数字**: 「あの求人、誰が持ってたっけ」がゼロ

### 2.2 PDF / 画像から AI 自動取り込み

- **何ができるか**: 求人票 PDF / 画像をドロップ → AI が構造化抽出
- **どう効くか**: 手入力ゼロ、求人登録が1分以内
- **数字**: 求人1件あたり10分 → 1分(90%削減)

### 2.3 CSV 一括インポート

- **何ができるか**: 大量求人を CSV で一括登録、AI 列マッピング対応
- **どう効くか**: 提携先からもらった求人リストを丸ごと取り込める
- **数字**: 100件の求人を5分以内で登録

### 2.4 AI マッチング(求人 × 求職者)

- **何ができるか**: 求職者の希望条件 × 自社求人 を AI が推薦
- **どう効くか**: アドバイザーの判断材料、ベテラン勘の補完
- **数字**: 推薦精度はベテランの初動感覚と同等、新人の即戦力化

## カテゴリ 3: 公式 LINE 連携

### 3.1 個別トーク

- **何ができるか**: LINE Messaging API 経由で個別トークを Maira 内で完結
- **どう効くか**: LINE 公式アプリと Maira を行き来する必要なし
- **数字**: アドバイザー1人あたりのトーク対応速度が向上

### 3.2 担当者割当 + タグ + メモ

- **何ができるか**: トーク相手に担当者・タグ・メモを紐付け
- **どう効くか**: 求職者情報とトークが完全リンク
- **数字**: 引き継ぎ時のコンテキスト共有が即完了

### 3.3 一斉配信(テキスト / 求人カード)

- **何ができるか**: タグや絞り込み条件で対象を絞って一斉配信
- **どう効くか**: 求人情報を関心の高い層に直接届ける
- **数字**: 配信効果(クリック率 / 応募率)を Maira 内で可視化

### 3.4 シナリオプリセット(7件標準)

- **何ができるか**: 「友達追加直後」「面談前」「面談後」「沈黙7日」等のシナリオ自動配信
- **どう効くか**: 求職者ジャーニーに沿った自動アプローチ
- **数字**: 配信工数を月10時間以上削減

### 3.5 友達追加時の自動メッセージ + Rich Menu

- **何ができるか**: 友達追加 → 自動応答 → ヒアリング誘導
- **どう効くか**: 初動の取りこぼしゼロ
- **数字**: 24時間以内の初回返信率 100%(自動)

### 3.6 クリック / 応募計測

- **何ができるか**: 配信した求人カードのクリック・応募を Maira 内で計測
- **どう効くか**: 「どの求人がどれだけ反応されたか」が数字でわかる
- **数字**: マーケ意思決定の高速化

## カテゴリ 5: AI 書類生成

### 5.1 会議録音 → AI 履歴書 / 職務経歴書

- **何ができるか**: 面談録音をアップロード → Whisper で文字起こし → Claude で構造化 → 履歴書 / 職経 PDF 生成
- **どう効くか**: ヒアリングから書類完成まで自動化
- **数字**: 書類1件あたり30分 → 5分(83%削減)

### 5.2 AI 推薦文ドラフト

- **何ができるか**: 求職者プロフィール → AI が推薦文を生成
- **どう効くか**: ベテランの推薦文クオリティを新人でも再現
- **数字**: 推薦文作成時間20分 → 2分

### 5.3 ヒアリングシート自動生成

- **何ができるか**: 録音から構造化されたヒアリングシートを自動作成
- **どう効くか**: 面談メモを手入力する必要なし
- **数字**: 面談後の事務処理時間15分 → 0分

### 5.4 推薦文テンプレート(組織共通)

- **何ができるか**: 冒頭挨拶 / 末尾定型句を組織で共通管理
- **どう効くか**: 推薦文の品質・トーンを統一
- **数字**: 推薦文の品質ばらつき解消

### 5.5 AI 証明写真補正

- **何ができるか**: 求職者の自撮り写真を証明写真風に AI 補正
- **どう効くか**: 求職者が証明写真を撮りに行く負担をなくす
- **数字**: 書類完成までのリードタイム短縮

## カテゴリ 10: セキュリティ

### 10.1 サーバーサイド AES-256-GCM 暗号化

対象フィールド:

- 会話履歴(messages.encrypted_content)
- キャリア棚卸し結果(career_profiles.encrypted_data)
- 応募情報(applications.encrypted_details)
- タスク内容(agency_tasks.encrypted_title, encrypted_description)
- 通知ペイロード(notifications.encrypted_payload)
- 会話タイトル(conversations.encrypted_title)
- LINE メッセージ(line_messages.encrypted_content)
- LINE 配信本文(line_broadcasts 系)

### 10.2 Supabase RLS(行レベルセキュリティ)

- 全テーブルで RLS を有効化
- `organization_id = current_user_organization_id()` で他組織のデータには物理的にアクセス不可

### 10.3 Webhook 署名検証(HMAC-SHA256)

- LINE / Stripe / Zoom の Webhook をすべて署名検証

### 10.4 プライバシーポリシー

- 運営者の閲覧範囲を「保管」「本人同意済みの AI 処理」「法令対応」に限定
- 登録時に同意取得

### 10.5 解約時データエクスポート

- CSV / JSON で全データをエクスポート可能(運営に依頼)
