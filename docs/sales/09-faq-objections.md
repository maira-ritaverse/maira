# FAQ + 反論への答え(営業即用)

商談で頻出する質問・反論 + その答え。営業が即使える形にまとめている。

## カテゴリ A: 価格 / コスト

### A1. 「月額が高い」

A. 月額の絶対値より「アドバイザー1人あたりの月給」と比較してください。
中堅プランの月額は、アドバイザー1人の月給(50万円)の数%です。
一方、Maira の効果はアドバイザー0.6人分の稼働 = 月給30万円相当を生み出します。

数字で見ると:

- 月額 15万円 ÷ アドバイザー5名 = 1人あたり月3万円
- 月3万円のコストで月10時間以上の業務削減 + 取りこぼし防止

→ 「Excel + LINE 公式で人件費を浪費するより、月3万円で効率化したほうが安い」

### A2. 「初期費用はかかりますか?」

A. 0円です。クラウド SaaS なのでサーバー購入や機器導入は不要。
データ移行も CSV インポートで自社で完結します(運営サポートあり)。

### A3. 「途中で解約はできますか?」

A. はい。月次プランなら翌月から、年次プランなら年度末から解約可能。
解約予約 → 期末まで利用可能 → 期末で失効 の流れです。
データは CSV / JSON でエクスポート可能(運営に依頼)。

### A4. 「AI 利用量で追加課金されるのが不安」

A. 各プランに月次の AI 利用上限を含めています。
通常運用では上限に到達することはほぼありません。
上限近くになると admin に通知され、必要なら高プランへアップグレード可能。

### A5. 「無料トライアルはありますか?」

A. 30日間のフル機能無料トライアル。カード登録なしで開始可能。
トライアル期間中に効果を実数値で確認してから本契約可能。

## カテゴリ B: 機能・性能

### B1. 「AI の精度は本当に大丈夫か?」

A. Anthropic Claude Sonnet 4.6 を採用。2026年時点で日本語処理は最高水準。
特に履歴書 / 職経のような構造化された文書生成は、ベテランエージェントの初稿レベルに到達。
AI 出力はあくまでドラフトで、エージェントが最終調整する前提で設計しています。

### B2. 「会議録音の文字起こし精度は?」

A. OpenAI Whisper を採用。日本語の文字起こし精度は90%以上。
専門用語(医療 / IT 等)の誤認識は、AI 構造化時に Claude が文脈で補正する設計。

### B3. 「公式 LINE 連携は本当に動くか?」

A. LINE Messaging API を正規利用。
Channel ID / Secret / Access Token を保存し、Webhook で双方向通信。
個別トーク + 一斉配信 + シナリオ自動化が標準搭載。

### B4. 「Google Calendar / Zoom 連携の信頼性は?」

A. 公式 OAuth フローで認可、API のスコープは最小限(calendar.events / meeting:write)。
Google / Zoom 側の API 障害時はリトライ + エラー通知を実装。

### B5. 「スマホで使えますか?」

A. PWA(Progressive Web App)対応。
ホーム画面に追加すればネイティブアプリのように使えます。
LINE 連携と Daily ダイジェストはスマホで完結可能。

### B6. 「カスタマイズはどこまでできる?」

A. 基本機能はそのまま使う前提(業界フローが標準実装)。
組織共通の推薦文テンプレート / カスタム項目 / タグ等の「設定レベル」のカスタマイズは可能。
深い機能拡張(独自フロー追加等)は要相談。

## カテゴリ C: データ・セキュリティ

### C1. 「データはどこに保存される?」

A. Supabase Northeast Asia (Tokyo) リージョン。国内法の適用範囲内。
全ての機密フィールドは AES-256-GCM でサーバーサイド暗号化。

### C2. 「他社の求職者データが見える可能性は?」

A. ありません。Supabase RLS で物理的に組織分離。
仮にアプリにバグがあっても、DB レベルで他組織のデータには触れない設計。

### C3. 「運営者は私たちのデータを見られますか?」

A. 暗号化されているため、アプリ機能を介さない限り平文は見られません。
プライバシーポリシーで「保管 / 本人同意済み AI 処理 / 法令対応」の3範囲に閲覧を限定しています。

### C4. 「AI に送ったデータは学習に使われませんか?」

A. Anthropic / OpenAI ともに、API 経由のデータを学習に使用しない設定で利用しています(Enterprise / Business 規約)。
データの一時保存は30日以内で削除されます。

### C5. 「漏洩した場合の補償は?」

A. SaaS 一般の規約に従い、契約金額を上限とする責任。
ただし暗号化 + RLS + Webhook 署名検証 + 監査ログで、漏洩リスクを最小化しています。

### C6. 「個人情報保護法は遵守されているか?」

A. はい。取得 / 利用 / 第三者提供のすべてで適法な根拠を確保。
開示 / 訂正 / 削除請求への対応窓口を設置(support@maira.pro)。

## カテゴリ D: 導入・運用

### D1. 「導入に何日かかりますか?」

A. 中堅企業で1〜2週間で本格運用開始。
初日: アカウント作成 + メンバー招待
1週目: データ移行 + 連携設定
2週目: 操作レクチャー + 通常運用開始

### D2. 「既存データの移行は大変ですか?」

A. CSV インポート + AI 列マッピングで簡単。
500名のデータ移行が30分以内。複雑な構造は運営サポートあり。

### D3. 「アドバイザーが使いこなせるか?」

A. UI はスプレッドシート + LINE のような直感的操作。
オンライン操作レクチャー(録画あり)+ マニュアル(NotebookLM 対応)を提供。
過去の導入企業では1週間以内に基本操作を習得。

### D4. 「既存ツールと並行運用は?」

A. 可能。新規顧客のみ Maira、既存は徐々に移行のパターンが推奨。
並行期間に上限なし。

### D5. 「サポート対応は?」

A. メールサポート(support@maira.pro、1営業日以内に一次返信)。
契約期間中はバグ報告 / 機能要望を継続的に受け付け。

### D6. 「アップデートで業務に影響は?」

A. 既存業務フローを破壊する変更はしません。
新機能は明示的な ON/OFF オプションとして提供。
重要な改訂時は事前通知 + 既存運用との両立期間を設けます。

## カテゴリ E: 拡張性 / 将来性

### E1. 「アドバイザーが増えたら対応できる?」

A. 中堅(50名規模)まで標準対応。
100名超の場合は別途要相談(プラン / 機能カスタマイズで対応可能)。

### E2. 「複数拠点での運用は?」

A. 1組織 = 1テナントで全拠点を統合運用可能。
拠点別の集計 / 担当アドバイザー別表示で実質的な拠点管理を実現。

### E3. 「グローバル展開した場合は?」

A. 現状は国内事業に最適化。
海外拠点を持つ場合は別組織として運用を推奨。
GDPR 等は将来計画。

### E4. 「他のシステムと連携できる?」

A. 既存連携: LINE Messaging API / Google Calendar / Zoom / Resend(メール)/ Stripe(課金)。
新規連携は要相談(月次の機能ロードマップで検討)。

### E5. 「将来的に機能が増える保証は?」

A. 月次の機能追加 + 顧客要望ベースの優先順位付け。
GitHub の Public Roadmap で計画を共有(将来予定)。

## カテゴリ F: 営業の現場で出やすい反論

### F1. 「うちは今のやり方で十分回ってる」

A. 「今のやり方」では、アドバイザー1人あたりの売上が頭打ちです。
人を増やさず売上を2倍にしたいなら、ツールで効率化するしかない。
Maira は「今のやり方の延長」ではなく、「業務フロー自体をアップグレード」する選択肢。

### F2. 「AI に頼ると質が下がるのでは?」

A. AI は「ドラフトを作る役」、最終判断はベテランエージェント。
書類作成時間が30分 → 5分に減れば、空いた25分でベテランが質を磨ける。
質が下がるどころか「ベテランが質に時間を使えるようになる」のが本質。

### F3. 「セキュリティが心配で個人情報を AI に送りたくない」

A. 暗号化 + RLS + AI 提供企業の学習除外設定 で多重防御。
AI に送らない設定も可能(機能 OFF)。送る機能は事前同意を取得済みの範囲のみ。

### F4. 「うちは新規顧客が少ないから Maira は要らない」

A. 既存顧客の取りこぼし防止だけでも効果大。
新規開拓よりも、既存の100名の中で「あの人最近連絡してないな」を減らす方が ROI 高い。

### F5. 「LINE 公式アカウントの標準機能で十分」

A. 標準機能には「個別トーク + 顧客情報の紐付け」「沈黙アラート」「業務メモ」がありません。
LINE 公式 + Excel で運用すると、結局アドバイザーが両方を見比べる工数が大きい。

### F6. 「導入後すぐに使いこなせないと困る」

A. 操作レクチャー + マニュアル + 30日無料トライアルで「使いこなせない場合は契約しない」選択肢が常にあります。
初期費用 0 + 30日無料 = 試して気に入ったら本契約というローリスクモデル。

### F7. 「他社で似たようなツールを使ってる」

A. 「似た」は機能リストでしょうか、実体験ですか?
無料トライアルで Maira の AI 書類生成 / 公式 LINE 統合 / Daily ダイジェストを体験してから判断してください。
特に「会議録音 → 履歴書」のスピード感は他社にはない体験です。

### F8. 「経営層に承認をもらうのが大変」

A. /roi ページで自社の数字を入れて試算 → 「年間 ¥XXX 万円効果」のレポートをそのまま経営層に提示できます。
契約後の効果報告も Maira のレポート機能で自動生成。

## カテゴリ G: 経営者向けの最終クロージング質問

### G1. 「この投資で、来年の売上はどれくらい上がりますか?」

A. /roi 試算ページで自社の数字を入れて即試算可能。
中堅企業(アドバイザー5名)で年間 2,000万円規模の効果が標準ライン。

### G2. 「他社事例はありますか?」

A. 個別の社名公開は控えていますが、業種別の効果データは提示可能。
無料トライアルで自社で効果を確認してから判断するのが一番確実です。

### G3. 「最低契約期間は?」

A. 月次プランなら最低契約期間なし。
年次プランは1年単位、契約期間中は月次相当の割引が適用。

### G4. 「商談を進めるには次に何をすればいい?」

A.

1. 無料トライアルを開始(その場でアカウント作成)
2. 主要メンバー2〜3名で1週間試用
3. ROI 試算 + 効果を経営層に共有
4. 良ければ本契約に移行

トライアル中の操作レクチャーも無料で提供します。

## クロージングの定型文

### 経営者層に対して

> 「人を増やさず売上を伸ばす」ための最もシンプルな投資です。
> アドバイザー1人あたりの稼働を2倍にする設計で、月額は人件費の数%。
> 30日無料トライアルで効果を実数値で確認してから判断できます。

### 現場アドバイザー層に対して

> 「LINE と書類作成と進捗管理が1画面で完結」する世界が、明日から使えます。
> 朝の Daily ダイジェスト1通で1日のタスクが見えて、面談調整は LINE で候補日を3つ送るだけ。
> 履歴書は会議録音をアップロードするだけ。
> 1週間使えば、もう前のやり方には戻れません。
