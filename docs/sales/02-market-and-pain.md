# 市場・業界課題と Maira 選定理由

## 転職エージェント業界の現状(2026 年)

### 市場規模

- 国内の人材紹介業界(転職エージェント)市場は年間 約1兆円規模
- 求職者数の増加と労働流動化により、エージェント1社あたりの取り扱い案件数は年々拡大
- 一方で、エージェント1人あたりが担当できる人数の上限は構造的に頭打ち(平均50〜80名)

### プレイヤー構造

- **大手総合型**: 大量採用 + 大量送客モデル(リクルート / マイナビ等)
- **中堅特化型**: 業界・職種に特化(IT / 医療 / 製造 / 営業 / 経営幹部 等)
- **小規模ブティック型**: 1〜5名の少数精鋭、ハイクラス / 専門特化

Maira の主戦場は **中堅特化型 + 小規模ブティック型** = 社員5〜50名規模のエージェント企業。

## 中堅エージェント企業が抱える6つの課題

### 課題 1: アドバイザー1人あたりの対応可能数が頭打ち

- 平均的なアドバイザーは月50〜80名の求職者を担当するのが限界
- LINE 個別対応・履歴書作成・面談調整に時間を取られ、新規対応に手が回らない
- 求職者数が増えても、人を増やさないと売上が伸びない構造

### 課題 2: 履歴書・職務経歴書の作成負荷

- 1件あたり30分〜1時間の作成時間(初稿)
- ヒアリングメモから手入力で書類を起こす作業が重い
- 求職者本人に書いてもらってもクオリティがばらつき、結局アドバイザーが書き直す
- 月100件の書類を扱うと、書類作成だけで月50時間以上を消費

### 課題 3: 連絡漏れによる機会損失

- 「あの求職者、最近連絡してないな」が頻発
- スプレッドシート管理では沈黙顧客が一覧化できない
- 月平均2〜5件の案件を取りこぼし、紹介料換算で月150〜400万円の機会損失

### 課題 4: 公式 LINE 運用の煩雑さ

- 求職者の8割以上が LINE 中心のコミュニケーションを希望
- 個別トーク + 一斉配信 + シナリオ自動化 + 友達リスト管理 が分散
- LINE 公式アカウント標準機能では、転職エージェント業務に最適化されていない

### 課題 5: 面談スケジュール調整の煩雑さ

- 候補日3つを LINE / メールで送る → 求職者返信待ち → カレンダー登録 → URL 発行
- 1件あたり10〜15分の調整コスト × 月50件 = 月8時間以上のロス
- 面談前のリマインダー漏れで無断キャンセル発生

### 課題 6: 進捗・成約状況の可視化不足

- アドバイザーごとの担当数 / 沈黙数 / 期限切れタスク数 が見えない
- 月末に手作業で集計、その時点では遅い
- 経営幹部が「今、誰が、何を、どこまで進めているか」を1画面で把握できない

## なぜ既存ツールでは解決できないのか

### 汎用 CRM(HubSpot / Salesforce / Pipedrive 等)

- **強み**: カスタマイズ性 / エンタープライズ向け機能
- **弱み**:
  - 転職エージェント業務に特化していない(求職者・求人・推薦という独自概念がない)
  - 月額10万円〜30万円以上、中堅企業には高い
  - カスタマイズに数週間〜数ヶ月、社内 SE が必要
  - 公式 LINE 連携は別ツール / 別契約
  - AI による書類自動生成機能なし

### 既存の人材紹介向け国内 SaaS

- **強み**: 業界特化、応募管理に強い
- **弱み**:
  - UI が古い(2000 年代設計のまま)
  - 公式 LINE 連携が後付け、機能が弱い
  - AI 機能が後追いで導入され、精度が不安定
  - モバイル対応(スマホでの操作)が弱い

### Excel / スプレッドシート + LINE 公式アカウント

- **強み**: 初期コスト0
- **弱み**:
  - データの一元管理ができない、共有が手作業
  - 沈黙顧客 / 期限切れタスクの可視化ゼロ
  - スプレッドシートが肥大化、検索性が落ちる
  - 公式 LINE のトーク履歴と顧客情報が分離

## Maira が選ばれる3つの理由

### 理由 1: 業界フローが標準実装されている

Maira は「クライアント(求職者) → 求人 → 紹介(referral) → 応募 → 面接 → 内定 → 成約」という業界の標準フローが画面構造そのものになっています。
カスタマイズ不要、その日から使えます。

### 理由 2: AI が業務の一部として組み込まれている

「AI 機能をつける」ではなく、「AI が前提の業務設計」になっています。

- 会議録音 → 履歴書・職経・推薦文を AI が自動生成
- 求人マッチング(求職者の希望条件 × 自社求人)を AI が提案
- ヒアリングシートを録音から自動構造化
- AI 証明写真補正(自撮り → 証明写真風)
- CSV 列マッピングの AI 推測

### 理由 3: 公式 LINE が深く統合されている

- 個別トーク + 担当者割当 + タグ + メモ + 編集履歴
- 一斉配信(テキスト / 求人カード)、シナリオプリセット7件
- 友達追加時の自動メッセージ・Rich Menu 等を設定
- LINE 経由で日程候補を一括提案 → 求職者選択で自動カレンダー登録
- クリック・応募計測で配信効果を可視化

## 想定顧客プロファイル

### 規模感

- 社員数: 5〜50名(うちアドバイザー3〜30名)
- 月間取り扱い求職者数: 50〜500名
- 月間成約数: 3〜30件
- 紹介料単価: 50〜150万円

### 業種特化の例

- **IT エンジニア専門**: 公式 LINE での頻繁なやり取りが多い、Maira の LINE 統合が即効
- **医療 / 看護師専門**: 求職者の希望条件が複雑、AI マッチングが効く
- **製造 / 工場長 / 技術者**: 履歴書 / 職経のフォーマット遵守が重要、AI 書類生成が即効
- **営業 / 経営幹部 / ハイクラス**: 推薦文のクオリティが成約に直結、AI 推薦下書きが効く

### 経営課題で言うと

- 「アドバイザー1人あたりの売上を上げたい」企業
- 「人を増やさず売上を伸ばしたい」企業
- 「データドリブンに経営判断したい」企業
- 「Excel 管理から脱却したい」企業
- 「AI を本気で業務に組み込みたい」企業

## 経営者へのメッセージ

中堅エージェントの経営において、最大のレバレッジは **「アドバイザー1人あたりの対応可能数」を物理的に増やすこと** です。

Maira は AI と業界特化機能で、アドバイザー1人を「2倍 速く」「2倍 多くの顧客に」 対応させます。

- 月100時間の業務削減 = アドバイザー0.6人分の余剰
- 取りこぼし防止 = 機会損失の根絶
- 経営ダッシュボード = 数字に基づく即時判断

「AI に追加投資する」のではなく、「業務に AI が組み込まれた SaaS に切り替える」のが、最も投資対効果の高い選択です。
