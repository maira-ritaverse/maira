# 導入ステップ / 期間 / 工数 / データ移行

## 導入全体の流れ(中堅企業の例)

```
Day 0: 申込み(無料トライアル開始)
       ↓
Day 1: 組織アカウント作成 + 主要メンバー招待
       ↓
Day 1-2: 既存データ移行(CSV インポート)
       ↓
Day 2-3: 公式 LINE / Zoom / Google Calendar 連携
       ↓
Day 4-7: 全アドバイザーへの操作レクチャー
       ↓
Week 2: 通常運用開始 + Daily ダイジェスト稼働
       ↓
Month 1 末: 初月効果を数字で確認
       ↓
Month 3: 副担当 / 組織マネジメント機能を活用
```

導入期間の目安: **1〜2週間で本格運用開始**

---

## ステップ別 詳細

### Step 1: 組織アカウント作成(15分)

1. `https://app.maira.pro/signup` で組織管理者としてサインアップ
2. メール確認 → ログイン
3. 組織名を入力(後から変更可能)
4. プラン選択(無料トライアル開始)

担当: 経営者 or 情シス
工数: 15分

### Step 2: メンバー招待(30分 〜 1時間)

1. `/agency/members` → 「メンバーを招待する」
2. メールアドレス + 初期役割(admin / advisor)を指定
3. 招待メールを送信 → 招待者がサインアップ → 自動で組織に参加

担当: 管理者
工数: メンバー数 × 5分(10名なら50分)

### Step 3: データ移行(1〜3時間)

#### 既存求職者のインポート

1. 既存スプレッドシート / Excel を準備
2. `/agency/clients` → 「CSV インポート」
3. AI が列マッピングを自動推測(必要なら手動調整)
4. プレビュー確認 → インポート実行

500名のデータ移行が30分以内

#### 既存求人のインポート

1. 求人票 PDF / 画像 を `/agency/jobs` でアップロード(AI 自動取り込み)
2. または CSV 一括インポート
3. ステータス(open / closed)を設定

担当: 管理者 or アドバイザー
工数: 求職者500名 + 求人50件で約2時間

### Step 4: 公式 LINE 連携(30分 〜 1時間)

#### 前提

- LINE 公式アカウントの Messaging API チャネルが必要
- LINE Developers Console での設定が必要

#### 手順

1. LINE Developers Console で Messaging API チャネルを作成
2. Channel ID / Channel Secret / Channel Access Token を取得
3. Maira `/agency/line/settings` で保存
4. Maira が生成した Webhook URL を LINE Developers に登録
5. Webhook 検証(「Verify」ボタン)で疎通確認
6. 友達追加時の自動メッセージ / Rich Menu 等を設定

担当: 管理者(LINE Developers アカウントを持つ人)
工数: 30分〜1時間

### Step 5: Zoom / Google Calendar 連携(各15分)

#### Zoom

1. `/agency/settings/integrations` → 「Zoom アカウントを連携する」
2. OAuth 認可フローで Zoom にログイン
3. meeting:write スコープを許可
4. 完了

#### Google(Calendar + Meet)

1. 同画面で「Google アカウントを連携する」
2. OAuth 認可で calendar.events スコープを許可
3. 完了

担当: アドバイザー本人(個人のアカウントと連携)
工数: 各15分

### Step 6: 推薦文テンプレート登録(30分)

1. `/agency/settings/recommendation-letter-templates`
2. 冒頭挨拶 / 末尾定型句を組織で共通登録
3. AI 推薦文生成時に自動適用される

担当: 管理者 + ベテランアドバイザー
工数: 30分

### Step 7: 操作レクチャー(各アドバイザー1〜2時間)

#### コンテンツ

1. クライアント一覧 / 詳細の操作
2. 公式 LINE トークの返信方法
3. 面談候補日の提案フロー
4. AI 履歴書 / 職経 / 推薦文の使い方
5. Daily ダイジェストの読み方
6. タスク管理

#### 形式

- 録画レクチャー(社内に保管)
- マニュアル(`docs/manual/` を NotebookLM に入れて AI 質問回答対応)
- 個別 OJT(各アドバイザー)

担当: ベテランアドバイザー or 運営サポート
工数: アドバイザー1人あたり1〜2時間

### Step 8: 通常運用開始

- Daily ダイジェストを毎朝確認
- 沈黙アラートを定期チェック
- 求人新規登録 → AI マッチングで候補抽出
- 面談実施 → 録音アップロード → AI 書類生成

## データ移行詳細

### 対応データ形式

- **CSV**: AI 列マッピング対応(列名が日本語でも自動推測)
- **Excel(.xlsx)**: CSV に変換してインポート(運営サポート可)
- **既存 SaaS からのエクスポート**: CSV 形式なら直接、独自形式は運営サポート

### マッピング可能な列

- 求職者: 氏名 / 氏名カナ / メアド / 電話 / 性別 / 生年月日 / 住所 / 希望職種 / 希望年収 / 状態 等
- 求人: 会社名 / 職種 / 雇用形態 / 勤務地 / 年収 / 必要スキル / 求人詳細 等

### 暗号化フィールドの扱い

- インポート時にサーバー側で AES-256-GCM 暗号化して保存
- 平文での DB 格納は一切しない
- 元 CSV ファイルは即削除推奨(運営側でも保管しない)

### 担当アドバイザーの紐付け

- CSV に「担当者メールアドレス」列を入れると、自動で組織メンバーに紐付け
- 不明な担当者は「未割り当て」として一時保管

- 客先での対面研修

## トラブル時の対応

### LINE が反応しない

1. `/agency/line/settings` で Channel Access Token が正しいか確認
2. Webhook URL が LINE Developers に登録されているか確認
3. Channel Secret が一致しているか(署名検証で失敗すると 401)

### Google 連携で「アクセスをブロック」が出る

- アプリが「テスト中」ステータスの場合、テストユーザー登録が必要
- 解決: Google Cloud Console → OAuth consent screen → テストユーザー → メアドを追加

### Daily ダイジェストが来ない

1. 個人設定 → 通知設定 で email_enabled + daily_digest が ON か確認
2. メール受信フォルダで「【Maira 朝のダイジェスト】」を検索
3. 集計が全件0の日は「平和な朝」として配信されません(仕様)

## 解約・データエクスポート

### 解約方法

- `/agency/settings/billing` から Stripe Customer Portal にリダイレクト
- 解約予約 → 期末まで利用可能 → 期末で失効

### データエクスポート

- 解約時に CSV / JSON で全データをエクスポート可能(運営に依頼)
- 求職者 / 求人 / 応募 / 推薦文 / 面談履歴 / LINE トーク全文 等

### データ保持期間

- 解約後30日間はデータを保持(再契約時に復元可能)
- 30日経過後に完全削除

## 他 SaaS との並行運用

- 移行期間中は Maira と既存ツールの並行運用が可能
- 段階的移行(まずは新規顧客のみ Maira、既存は徐々に移行)も推奨パターン
- 並行期間の長さに上限なし
