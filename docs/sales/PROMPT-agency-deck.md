# 営業提案資料 生成プロンプト(転職エージェント企業向け)

このファイルは、`docs/sales/` の 9 本のソース + `10-vision-and-philosophy.md`
を元に、長文営業資料(1 本の Markdown、15,000〜25,000 字、Mermaid 図・
イラスト指示コメント入り)を LLM に生成させるための指示書です。

## 使い方

1. Claude / GPT 等の LLM に、`docs/sales/` 配下のすべての `.md` を添付
   (01〜09 と `10-vision-and-philosophy.md`)
2. 下の「プロンプト本文」をそのまま貼って送信
3. 出力された Markdown を `docs/sales/generated/YYYY-MM-agency-deck.md`
   等の作業ディレクトリに保存
4. 必要に応じて Pandoc で PDF 化 / Marp でスライド化 / Google Docs に貼付

---

## プロンプト本文(このセクション全体をコピーして LLM に渡す)

````
# 役割
あなたは B2B SaaS の営業提案資料を専門とするシニアコピーライター兼
インフォグラフィックデザイナーです。企業説明会や商談で経営者が 10 分で
意思決定できる長文提案資料を作れます。読者への敬意と、業界構造への
深い理解を両立させることに強みがあります。

# 最終ゴール
株式会社 Revorise が提供する SaaS「Myaira」を、中堅・小規模の転職
エージェント企業(社員 5〜50 名)に導入してもらうための、長文営業
提案資料を 1 本の Markdown ファイルとして出力してください。分量は
15,000〜25,000 字。読者はこれを PDF 化して商談に持参します。

# 必読ソース(順序どおりに完全に読み込むこと)
1. docs/sales/01-executive-summary.md
2. docs/sales/02-market-and-pain.md
3. docs/sales/03-features-detail.md
4. docs/sales/04-pricing-roi.md
5. docs/sales/05-implementation.md
6. docs/sales/06-security-compliance.md
7. docs/sales/07-use-cases.md
8. docs/sales/08-competitive.md
9. docs/sales/09-faq-objections.md
10. docs/sales/10-vision-and-philosophy.md ← 「思い」原稿(必須逐語組込)

上記に書かれていない機能・数字を勝手に追加しないでください。曖昧な
部分は「〜と想定」「〜を目安」と留保付きで記述してください。

# 必須挿入:Myairaの思い(逐語ベース)
`10-vision-and-philosophy.md` は Revorise 代表からの原稿です。
第 2 章「なぜ Myaira か — 私たちの思い」として、語尾を本文に馴染ませる
以外は改変せずに全文を組み込んでください。要約・AI 風の言い換え・
順序入れ替えは禁止です。

# 想定読者と読み方
- 主読者:中堅転職エージェント企業の経営者・執行役員(意思決定者)
- 副読者:現場マネージャー / 情シス / 財務担当
- 想定シーン:
  - 経営者は 10 分眺めて「導入する価値があるか」を判断
  - マネージャーは「自社オペレーションでどう動くか」を想像
  - 情シスはセキュリティ章だけ抜いて評価

# 章構成(この順序・粒度で固定)
0. 表紙(タイトル / サブタイトル / 対象読者 / 発行:株式会社 Revorise / 版)
1. エグゼクティブサマリー(数字中心、1〜2 ページ相当)
2. なぜ Myaira か — 私たちの思い(★原稿を全文組み込み★)
3. 転職エージェント業界の現在地と 6 つの構造課題
4. Myaira が解決すること — 全体像
5. 差別化 3 要素の深堀り
   - 5-A プロアクティブな伴走(Daily ダイジェスト / 沈黙アラート)
   - 5-B 会議録音 → AI 履歴書 / 職経 / 推薦文 生成
   - 5-C 公式 LINE 連携 + 配信 / MA
6. 業種別ユースケース(IT / 医療 / 製造 / ハイクラス)
7. セキュリティ・コンプライアンス(AES-256-GCM / RLS / プライバシー)
8. 価格・ROI(規模別試算 2〜3 パターン)
9. 導入ステップとカスタマーサクセス体制
10. FAQ と反論への回答
11. 競合比較(汎用 CRM / 国内特化 SaaS / Excel 運用)
12. Revorise のお約束と次のステップ

# 章ごとのお作法
- 章冒頭に「この章で伝えたい 1 文」を太字で置く
- 「導入 → 本文 → 図 or 表 → まとめ 2〜3 行」の順で構成
- 章末に「読者への問い」または「次章への橋渡し」を 1 行

# トーン
- 一人称は「私たち」(Revorise として)
- エージェントを「置き換える」ではなく「主役として引き立てる」姿勢
- 業界へのリスペクトを随所ににじませる
- 敬体(です・ます調)
- 過度な誇張禁止(「圧倒的」「唯一無二」「業界 No.1」「革命的」は使わない)
- 数字を出すときは根拠(月◯件 × 時給◯円 のように)を必ず添える
- 「求職者の満足 = エージェントの成長」を通奏低音として全編で響かせる

# スタイル制約(絶対)
- 絵文字は一切使わない(章タイトル・箇条書き・コメントすべて)
- 「TODO」「後で書く」等のプレースホルダー禁止
- 「一般に」「よく言われている」など曖昧な出典表現禁止
- Myaira の実装済み機能のみを紹介(必読ソースに書かれていないものは書かない)

# Mermaid 図(最低 5 枚、以下の位置には必ず配置)
各図の直下に「読み取り: 〜」の 1 行キャプションを添えてください。

- 第 3 章:業界構造課題の因果フローチャート(flowchart LR)
- 第 4 章:Myaira の全体像(flowchart TD、求職者 / アドバイザー / AI / 外部連携)
- 第 5-B 章:会議録音から書類生成までのシーケンス図(sequenceDiagram)
- 第 8 章:ROI の因果ダイアグラム(flowchart LR、投入 → 削減 → 増収)
- 第 9 章:導入タイムライン(gantt)

```mermaid
sequenceDiagram
    participant U as 求職者
    participant M as Myaira
    participant A as アドバイザー
    ...
```

# イラスト指示コメント(最低 8 箇所)
イラストで理解が深まる箇所に、HTML コメントで指示だけ残してください。
後日デザイナーが差し替えます。形式は以下で統一:

```
<!-- ILLUSTRATION
  place: 第◯章 見出しの直下
  scene: (どんな絵か 1〜2 文で)
  intent: (何を伝えたいか)
  style: 温かみのある水彩調 / 手描き線
-->
```

配置候補:表紙 / 思い章冒頭 / 業界課題章 / ユースケース各業種 /
5 年後の未来像 / 章末エンディング

# 表について
- 数値比較・機能一覧は Markdown 表で
- 表の直後に「読み取り: 〜」の 1 行を必ず添える

# 禁止事項
- 実装されていない機能を約束する記述
- 実企業名・実顧客名を出すこと(必要なら「A 社(仮名)」と明示)
- 「AI がエージェント業務を代替する」といったエージェント不要論的な表現
- 競合を貶す表現(事実ベースで違いを述べるに留める)
- Revorise 代表の年齢や個人的属性への言及(会社としてのメッセージに徹する)

# 出力形式
1 本の Markdown ファイル。冒頭に以下の front matter を置いてください。

```yaml
---
title: Myaira 営業提案資料 — 転職エージェント企業向け
subtitle: 求職者に本気で寄り添うための、AI パートナー
audience: 中堅・小規模 転職エージェント企業(社員 5〜50 名)
publisher: 株式会社 Revorise
version: 2026-07 版
---
```

# 執筆後の自己チェック(必ず実施し、末尾に自己申告)
- [ ] 思い章が原稿の熱量を保ったまま全文組み込まれている
- [ ] 数字にはすべて根拠が添えられている
- [ ] 各章冒頭に「1 文サマリー」が太字で置かれている
- [ ] Mermaid 図が 5 枚以上入っている
- [ ] ILLUSTRATION コメントが 8 箇所以上入っている
- [ ] 絵文字ゼロ
- [ ] 「置き換え」ではなく「エージェントを主役に立てる」姿勢が貫かれている
- [ ] 全体 15,000〜25,000 字の範囲に収まっている

以上を守って、本編を出力してください。
````

## 補足:出力後の変換レシピ

### PDF 化(Pandoc)

```bash
pandoc docs/sales/generated/2026-07-agency-deck.md \
  -o out/2026-07-agency-deck.pdf \
  --pdf-engine=xelatex \
  -V mainfont="Hiragino Sans" \
  --toc
```

Mermaid 図は Pandoc 単体ではレンダリングされないため、事前に
`mermaid-filter` を挟むか、Typora / Obsidian で PDF 出力するのが早い。

### スライド化(Marp)

章単位で `---` 区切りを追加すれば Marp のスライドに変換可能。
経営者向けサマリー版は 01 章 + 02 章 + 12 章のみ抽出して 10 枚に。

### イラスト差し込み

`<!-- ILLUSTRATION -->` コメントが入っている箇所をデザイナーに渡し、
差し替え後の画像パスを `![](../assets/illust/xx.png)` 形式で置換。

## 更新方針

- 機能追加・数字更新が発生した場合、まず該当する 01〜09 の
  ソース .md を更新
- 「思い」を書き直す場合は `10-vision-and-philosophy.md` を更新
- プロンプト自体の改良は本ファイルを更新し、次回生成時に反映
