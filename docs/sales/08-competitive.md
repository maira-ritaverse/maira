# 競合比較

エージェント企業が検討する代表的な選択肢と Maira の比較。
営業の場では、相手が現在使っているツールに合わせて該当セクションを抜粋する。

---

## 大カテゴリ別の選択肢

| カテゴリ                             | 代表例                           | 強み                                     | 弱み                                      |
| ------------------------------------ | -------------------------------- | ---------------------------------------- | ----------------------------------------- |
| 汎用 CRM                             | HubSpot / Salesforce / Pipedrive | カスタマイズ性 / 機能網羅                | 業界特化なし / 高価格 / 導入長期化        |
| 国内エージェント特化 SaaS            | 各社の業界向け CRM               | 業界フローに合致                         | UI 古い / AI 弱い / LINE 連携弱い         |
| Excel / スプレッドシート + LINE 公式 | 自社運用                         | 初期コスト 0                             | 規模拡大不能 / データ分散 / 取りこぼし    |
| **Maira**                            | —                                | **業界特化 + AI ネイティブ + LINE 統合** | 大規模(100名超)エージェントには別途要相談 |

| 業界特化                   | ✅ 転職エージェントに完全特化                 | ❌ 汎用、業界ごとに大規模カスタマイズ必要                                 |
| 公式 LINE 連携             | ✅ 標準搭載、配信 / MA 統合                   | ❌ なし(別ツール / 別契約)                                                |

| AI 推薦文                  | ✅ Claude Sonnet 4.6 で生成                   | ❌ ChatBot 機能はあるが推薦文生成は別途                                   |
| Daily ダイジェスト         | ✅ 標準搭載                                   | ❌ Workflow で類似は可能だが業界向けは独自構築必要                        |
| 沈黙アラート               | ✅ 30/60/90日で自動                           | ❌ Workflow 独自構築                                                      |

- 業界フローがそのまま使える、カスタマイズ不要

## 切り替え時の比較ポイント(営業の場で聞かれる質問)

### Q1. データ移行は大丈夫か?

A. CSV インポート + AI 列マッピングで500名クラスを30分以内に移行可能。
複雑な構造のデータも運営サポートで対応。

### Q2. 既存ツールと並行運用は可能か?

A. 可能。 「新規顧客のみ Maira」「既存は徐々に移行」 のパターンが推奨。
並行期間に上限なし。

### Q3. AI の精度は他社と比べてどうか?

A. Anthropic Claude Sonnet 4.6 は2026年時点で最高水準のモデル。
特に日本語の文章生成 / 構造化は GPT-4 系よりも安定して品質が高いと評価される。

### Q4. アドバイザーが使いこなせるか?

A. UI はスプレッドシート同等の使い慣れた形式 + LINE のような直感的操作。
オンライン操作レクチャー + マニュアル(NotebookLM 取り込み可)を提供。
過去の導入企業では、ベテランから新人まで1週間以内に基本操作を習得。

### Q5. データはどこにあるか?

A. Supabase の Northeast Asia (Tokyo) リージョン。国内法の適用範囲内。
日次バックアップ + Point-in-Time Recovery 対応。

### Q6. 解約時のデータはどうなるか?

A. CSV / JSON で全データをエクスポート可能(運営に依頼)。
解約後30日間データ保持 → 完全削除。

### Q7. 大手 SaaS と比べて開発元の規模は?

A. 株式会社Revorise(国内)の運営。
チームが小さい分、機能要望への対応速度が速い(数週間で機能追加されることが多い)。

### Q8. SOC 2 / ISO 27001 は取得しているか?

A. 現状未取得。ただし基礎要件(暗号化 / RLS / 監査ログ / バックアップ / RBAC)は実装済。
顧客の要請があれば取得計画を加速する想定。

## Maira が「合わない」ケース

正直に提示すべきケース:

### 大規模(100名超のアドバイザー)エージェント

- 現状は中堅向け設計
- 100名超の規模は別途要相談(プラン / 機能カスタマイズが必要)

### 高度なカスタマイズが必要な場合

- Salesforce 級の自由度は持たない
- 業界フローはカバーするが、独自の特殊フローは別途相談

### グローバル展開(海外法人)

- 現状は国内事業に最適化
- GDPR 等の地域別法令対応は将来計画

### 完全オンプレ要求

- クラウド SaaS のみの提供
- オンプレや専有クラウドは現状非対応

## 切り替えタイミングの提案

### 切り替えるべきサイン

1. アドバイザー1人あたりの担当数が頭打ち(50名前後)
2. 月の取りこぼし案件が3件以上
3. 書類作成時間が月50時間を超える
4. 経営者が「今、誰が何やってるか」が即答できない
5. 公式 LINE の対応が他のツールと分断している

### 切り替えのベストタイミング

- 期初(4月 / 10月)= 業務フロー変更を組織で吸収しやすい
- 大型案件の谷間 = 移行期間に余裕がある
- 増員予定の直前 = 新メンバーは Maira 前提で教育できる
