# 業種別 ユースケース

業種ごとに「どの機能がどう効くか」を具体的なシナリオで提示。
営業の場では、相手の業種に合わせて該当セクションを抜粋して使う。

---

## ユースケース 1: IT エンジニア専門エージェント

### プロファイル

- アドバイザー数: 5〜15名
- 月間求職者: 100〜300名
- 平均紹介料: 80〜120万円
- 求職者の特性: 20-40代、男女比 8:2、LINE 中心、選考スピード重視

### 主な業務課題

- 求職者の95%が LINE 中心、メールはほぼ見ない
- 選考スピード(初動から内定まで2-4週間)が他業界より速く、リアルタイム対応必須
- スキルセット(言語 / フレームワーク / 経験年数)が複雑、マッチングが難しい

### Myaira での解決

#### LINE 統合が即効

- 個別トーク + タグ + メモが Myaira 内で完結
- 「Python 経験者」「フロントエンド希望」等のタグで瞬時に絞り込み一斉配信
- 友達追加直後の自動ヒアリング → 求職者情報を初動で取得

#### AI マッチングで初動が速い

- 求職者が希望条件を伝えた瞬間、自社求人から AI が候補を3〜5件抽出
- 担当アドバイザーが LINE で即送信
- 「求人を探す時間」がゼロに

#### 推薦文 AI が新人を即戦力化

- ベテランの推薦文クオリティを新人でも再現
- IT 用語(言語 / フレームワーク等)を正しく扱う AI が必須
- Claude Sonnet 4.6 が技術用語を高精度で扱う

### 想定 ROI

平均 ROI 倍率 × 1.2(LINE 効果が大)
中堅10名規模で年間 約2,500万円効果

## ユースケース 2: 医療 / 看護師専門エージェント

### プロファイル

- アドバイザー数: 8〜20名
- 月間求職者: 150〜400名
- 平均紹介料: 70〜100万円
- 求職者の特性: 20-50代、女性比率高、勤務形態(夜勤/日勤/オンコール)・通勤・子育て両立など希望条件が多次元

### 主な業務課題

- 求職者の希望条件が多次元(年収・勤務地・夜勤の有無・託児所・通勤時間 等)で、ベテランでないと絞り込みが難しい
- 看護師の働き方が多様化(常勤 / 派遣 / 訪問看護 / 美容医療等)、求人ごとに必要な書類が違う
- 求職者は複数エージェントに同時登録するのが普通、初動スピードが成約に直結

### Myaira での解決

#### AI マッチングで多次元条件を瞬時に評価

- 希望条件(勤務形態 / 通勤 / 託児所等)を AI が一括評価
- ベテランの「あの求人が合いそう」勘を新人でも再現
- 求人と求職者のマッチ度を数値で表示

#### 副担当機能で複雑なケースを共同対応

- 子育て両立希望 → ベテラン女性アドバイザーを副担当に
- 夜勤専門希望 → 病院系を持つアドバイザーを副担当に
- 主担当が知らない情報を副担当が補完

#### Daily ダイジェスト + 沈黙アラートで初動高速化

- 複数エージェント同時登録 → 早く動いた方が勝つ
- 朝の Daily ダイジェストで「今日連絡すべき新規」を1通で把握
- 沈黙アラートで「他社に流れる前に」フォロー

### 想定 ROI

平均 ROI 倍率 × 1.15(マッチング + 副担当の効果が大)
中堅15名規模で年間 約3,500万円効果

## ユースケース 3: 製造 / 工場長 / 技術者専門エージェント

### プロファイル

- アドバイザー数: 3〜10名
- 月間求職者: 50〜150名
- 平均紹介料: 80〜150万円
- 求職者の特性: 30-60代、男性比率高、技術スキル詳細、転職経験少なめ

### 主な業務課題

- 求人票が紙 / PDF / FAX で来る(企業側のデジタル化が遅れている)
- 求職者の職務経歴書が文章として書きにくい(技術 / 工程経験を体系的に書く必要)
- 工場長クラスは「他社に転職することを誰にも言えない」秘匿性が必要

### Myaira での解決

#### PDF / 画像から AI 自動取り込み

- 紙 / FAX で来た求人票を写真撮影 → アップロード → AI が構造化
- 手入力ゼロ、登録1分以内

#### AI 履歴書 / 職経自動生成

- 工場長 / 技術者の経歴は構造化しにくい
- 会議録音(面談)を AI が文章として整える
- 「○○工場で○年勤務、××ラインの責任者として△△を改善し...」等を自動生成

#### 秘匿性のための暗号化

- 求職者の経歴は AES-256-GCM で暗号化保存
- 「他社に転職検討中」を社外に漏らさない安心感
- 求職者本人にもプライバシーポリシーで明示

### 想定 ROI

平均 ROI 倍率 × 1.25(書類生成効果が大)
中堅5名規模で年間 約2,800万円効果

## ユースケース 4: 営業 / 経営幹部 / ハイクラスエージェント

### プロファイル

- アドバイザー数: 3〜8名
- 月間求職者: 30〜80名(少数精鋭)
- 平均紹介料: 150〜500万円
- 求職者の特性: 30-50代、年収800万円〜2,000万円、キャリア戦略重視

### 主な業務課題

- 求職者1人あたりの対応時間が長い(初回面談2時間、その後の壁打ちも含めて月数時間)
- 推薦文のクオリティが内定に直結(経営層は推薦文を熟読)
- 求人案件数は少ないが、1件あたりの紹介料が大きい
- 求職者本人も慎重(競合他社への情報漏洩を懸念)

### Myaira での解決

#### AI 推薦文の高クオリティ生成

- 求職者プロフィール → Claude Sonnet 4.6 が推薦文を生成
- 組織共通テンプレート(冒頭挨拶 / 末尾定型句)で品質統一
- ベテラン推薦文のスタイルを学習させて 80% 完成度のドラフトを出す

#### 会議録音 → ヒアリングシート自動構造化

- 2時間の面談録音を AI が要約 + 構造化
- 強み / 弱み / 希望 / 過去実績 を整理
- 推薦文作成の元データとして直接使える

#### 副担当機能で複数視点の意見

- 経営層の転職判断は複数人の意見が必要
- ベテラン2-3名で1求職者を担当
- 各自の見立て / 推奨ポジションを Myaira 内で共有

#### 暗号化 + RLS で秘匿性確保

- 経営層の転職検討は社外に漏らせない
- 暗号化 + 組織分離で物理的に保護

### 想定 ROI

平均 ROI 倍率 × 1.3(推薦文クオリティ + 副担当の効果が大)
小規模5名規模でも年間 約4,000万円効果(紹介料が高いため)

## ユースケース 5: 新卒 / 第二新卒専門エージェント

### プロファイル

- アドバイザー数: 5〜15名
- 月間求職者: 200〜600名
- 平均紹介料: 30〜80万円
- 求職者の特性: 20-25代、デジタルネイティブ、LINE / Instagram 中心

### 主な業務課題

- 求職者数が多い(月500名級)、1人1人に深くは関われない
- 説明会・選考対策セミナーの一斉案内が必要
- 新卒は「決断スピード」が遅い(複数社並行)、長期フォロー必須

### Myaira での解決

#### LINE 一斉配信 + シナリオ自動化

- 「3月卒業予定」「営業職希望」等のタグで対象を絞り込み
- 説明会 / セミナー案内を一斉配信
- 「友達追加 → 1週間 → 7日後フォロー → 14日後リマインド」等を自動化

#### Daily ダイジェストで大量管理を可能に

- アドバイザー1人で 100名超を担当しても、朝の1通でやることが見える
- 沈黙アラートで「2週間放置」を即発見

#### 求人カード配信 + クリック計測

- 新卒求人を求人カード形式で一斉配信
- クリック率 / 応募率で「効いた配信」を可視化
- 配信戦略を数字で最適化

### 想定 ROI

平均 ROI 倍率 × 1.1(LINE 配信 + 大量管理の効果)
中堅10名規模で年間 約2,200万円効果

## ユースケース 6: 副業 / フリーランスエージェント

### プロファイル

- アドバイザー数: 2〜5名(少数)
- 月間求職者: 30〜80名
- 平均紹介料: 50〜200万円(案件単発)
- 求職者の特性: スキル特化、複数案件並行が前提

### 主な業務課題

- 1求職者が複数案件を並行 → 案件単位の進捗管理が複雑
- 「いつ稼働開始できるか」が成約条件
- フリーランス契約特有の書類(NDA / 業務委託契約 等)が多い

### Myaira での解決

#### 応募管理の状態遷移で複数案件を可視化

- 1求職者 × N 案件の応募を Myaira 内で並列管理
- 各案件のステージ(打診中 / 面談中 / 契約交渉中 / 稼働中)を一覧表示

#### 副担当機能で案件 / 求職者の両面サポート

- 案件側担当(企業との折衝)+ 求職者側担当(キャリア相談)を分けて管理
- 役割分担を明示

#### スキル / 領域タグで瞬時に絞り込み

- 「React / TypeScript」「データ分析 / Python」等のタグ
- 案件発生時に即マッチ候補を抽出

### 想定 ROI

平均 ROI 倍率 × 1.2(応募管理 + マッチング効果)
小規模3名規模で年間 約1,500万円効果

## 業種×ユースケース 推奨機能マッピング

| 業種                | 最重要機能                 | 次点                      |
| ------------------- | -------------------------- | ------------------------- |
| IT エンジニア       | LINE 統合 + AI マッチング  | Daily ダイジェスト        |
| 医療 / 看護師       | AI マッチング + 副担当     | LINE 配信                 |
| 製造 / 技術者       | AI 書類生成 + PDF 自動取込 | 暗号化(秘匿性)            |
| 営業 / 経営幹部     | AI 推薦文 + 会議録音       | 副担当(複数視点)          |
| 新卒 / 第二新卒     | LINE 一斉配信 + シナリオ   | Daily ダイジェスト        |
| 副業 / フリーランス | 応募状態遷移 + マッチング  | 副担当(案件 / 求職者分担) |

営業の場では、相手の業種にあった「最重要機能」を中心にデモを設計すると効果的。
